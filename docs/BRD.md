## 一、项目背景

### 1.1 市场现状

中国同城货运市场规模约 **1,500 亿元**（2023 年数据），预计年均增速 8-10%。主要参与者包括货拉拉、滴滴货运、快狗打车等，市场呈现"一超多强"格局——货拉拉占据约 50% 份额，滴滴货运依托集团生态稳居第二梯队。

行业核心特征：

- **低频刚需**：个人用户平均月发单 1-2 次，B 端用户频次更高但黏性弱
- **供需高度分散**：司机端和货主端都是长尾分布，调度效率是核心竞争力
- **补贴驱动增长**：各平台获客高度依赖补贴，补贴占新客获取成本的 40-60%
- **留存是核心挑战**：D30 留存率普遍在 20-30%，大量用户首单后流失

### 1.2 滴滴货运的定位与挑战

滴滴货运 2020 年入场，核心优势在于滴滴集团 **5,000 万 DAU 的出行生态**。通过交叉导流从出行用户池中筛选货运意向用户，理论上获客成本（CPA）可大幅低于纯货运平台。但实际运营面临严峻挑战：

- 集团用户池中货运意向用户占比不足 5%，精确识别难度大
- 出行→货运的跨品类转化链路长、损耗高
- 补贴 ROI 持续下降（2021 年 ~3.5x → 2023 年 ~1.8x）
- 竞品持续加码，用户多端比价行为增加

### 1.3 为什么需要 AI 驱动的增长系统

传统增长运营面临"三高"困境：

| 维度 | 现状 | 理想状态 |
|------|------|----------|
| 人力成本 | 10+ 人增长团队手动分析 | 1-2 人 + AI 系统自动分析 |
| 决策速度 | 周级别策略迭代 | 日级别甚至实时调整 |
| 数据利用率 | <20% 的数据被有效利用 | >80% 的数据驱动决策 |

GrowthPilot Agent 正是为解决这一矛盾而生——用 Multi-Agent 架构将增长专家的经验系统化，将数据分析从"人驱动"升级为"AI 驱动 + 人审核"。

## 二、业务问题定义

### 2.1 五大核心痛点

#### 痛点 1：潜客识别效率低

**现状**：
- 传统规则模型（基于用户画像标签 + 人工阈值）AUC 仅为 0.62-0.68
- 日均需从 3,000 万出行用户中筛选约 15 万高潜用户，命中率不足 8%
- 模型迭代周期长达 1-2 个月，无法快速适应市场变化

**影响**：
- 营销预算浪费在低意向用户身上
- 高潜用户被遗漏，竞品率先触达

**量化估算**：
- 意向模型 AUC 每提升 0.05，每月可节约营销预算约 ¥15-20 万
- 高潜用户识别率从 8% 提升至 12%，月增量订单可提升 15-20%

#### 痛点 2：补贴浪费严重

**现状**：
- 平均补贴 ROI 仅 1.8x（即投入 ¥1 补贴产出 ¥1.8 增量 GMV）
- 约 35% 的补贴发放给了"本就会转化"的自然转化用户（deadweight loss）
- 补贴面额设置依赖经验（统一发 ¥10/¥20 券），缺乏个性化

**影响**：
- 月均 ¥500 万的补贴预算中，约 ¥175 万属于无效支出
- 用户产生补贴依赖，补贴一旦收紧订单量骤降

**量化估算**：
- 通过因果推断精确识别补贴敏感用户，ROI 可提升至 2.5-3.0x
- 个性化发券（基于价格弹性分层），可在同等预算下提升 25-30% 增量订单

#### 痛点 3：流失率居高不下

**现状**：
- D30 留存仅 25%（即 100 个新客中 75 个在 30 天内流失）
- D7 留存约 40%，说明大量用户在首周就已流失
- 流失原因分析滞后，通常在流失发生 2-4 周后才得出结论

**影响**：
- 高获客成本（CAC ~¥180）在低留存率下被放大
- 用户 LTV 降低，获客-变现模型失衡

**量化估算**：
- D30 留存每提升 5 个百分点，月增量 GMV 约 ¥80-120 万
- 提前 7 天识别高危用户并干预，挽回率可达 15-20%

#### 痛点 4：端外投放 CPA 高

**现状**：
- 域外广告（抖音/快手）平均 CPA ¥80+，且质量参差不齐
- RTA（Real-Time API）过滤规则粗放，约 30% 的竞价为无效出价
- 创意同质化严重，CTR 持续下降（季度环比 -15%）

**影响**：
- 月均 ¥200 万的广告预算效率持续走低
- 广告带来的新客 LTV 偏低（~¥120），难以覆盖获客成本

**量化估算**：
- RTA 策略优化（精确过滤 + 个性化出价）可将 CPA 降低 20-30%
- 创意疲劳检测及时替换素材，CTR 可回升 10-15%

#### 痛点 5：决策靠经验而非数据

**现状**：
- 增长策略"拍脑袋"现象严重，60% 以上的策略未经 A/B 测试验证
- 缺乏统一的实验平台，实验设计不规范、样本量不足
- 策略复盘依赖手工 SQL + Excel，周期长且容易出错

**影响**：
- 策略效果无法量化，持续优化缺乏依据
- 重复犯同样的错误（如：暑期补贴加量后 ROI 反而下降）

**量化估算**：
- 标准化 A/B 实验可将策略迭代效率提升 3-5 倍
- 实验驱动的决策相比经验驱动，平均效果提升 20-40%

## 三、解决方案概述

### 3.1 Multi-Agent 系统架构

GrowthPilot Agent 采用 **Multi-Agent + LangGraph StateGraph** 架构，将增长全生命周期拆分为 5 个专业 Agent 和 1 个编排 Agent：

```
┌─────────────────────────────────────────────────┐
│              Orchestrator Agent                  │
│  (意图识别 → 任务调度 → 结果聚合 → 策略生成)     │
└──────┬──────┬──────┬──────┬──────┬──────────────┘
       │      │      │      │      │
  ┌────▼──┐ ┌─▼───┐ ┌▼────┐ ┌▼────┐ ┌▼──────────┐
  │Prospect│ │Subsidy│ │ Ad │ │Conv.│ │ Retention │
  │Agent  │ │Agent │ │Agent│ │Agent│ │  Agent    │
  └───────┘ └──────┘ └─────┘ └─────┘ └───────────┘
```

### 3.2 各 Agent 的业务价值

| Agent | 解决的痛点 | 核心能力 | 预期业务价值 |
|-------|-----------|---------|-------------|
| **ProspectAgent** | 痛点 1 | LightGBM 意向模型 + RFM 分群 + LTV 预测 | AUC 提升至 0.75+，高潜命中率提升 50% |
| **SubsidyAgent** | 痛点 2 | DoWhy 因果推断 + 价格弹性 + 预算优化 | ROI 从 1.8x 提升至 2.8x，浪费减少 40% |
| **RetentionAgent** | 痛点 3 | 流失预警 + 新客养成 + 同期群分析 | D30 留存提升 5pp，挽回率提升 15% |
| **AdAgent** | 痛点 4 | RTA 策略 + 出价优化 + 创意分析 | CPA 降低 25%，CTR 回升 12% |
| **ConversionAgent** | 痛点 2/5 | 漏斗分析 + 券策略 + 资源位分配 | 转化链路效率提升 20% |
| **Orchestrator** | 痛点 5 | 意图识别 + 全局策略综合 + KPI 聚合 | 决策效率提升 5 倍 |

### 3.3 技术创新点

1. **精确计算 + AI 解读**：Python 工具（NumPy/Pandas/LightGBM/DoWhy）负责精确数值计算，LLM 负责策略生成和业务解读——既保证计算准确性，又发挥 LLM 的语义理解优势
2. **因果推断替代相关性分析**：通过 DoWhy 框架识别补贴的真实因果效应，而非仅看相关关系，避免"给本就会转化的用户发补贴"
3. **多模型分层调用**：按任务复杂度选择不同 LLM tier（fast/default/power），在保证质量的同时降低 40-60% 的调用成本
4. **渐进式降级**：每个 Agent 的工具调用都有 heuristic fallback，单工具失败不影响整体分析完成

## 四、业务指标体系

### 4.1 北极星指标

**MAO（Monthly Active Order users）**——月活跃下单用户数

> 选择理由：MAO 综合反映了获客效率（新客贡献）、留存质量（老客贡献）和平台价值（用户愿意付费），是增长全链路效果的终极体现。

```
MAO（北极星）

│   ├── L2: 端内新客数

│   │   ├── L3: 触达覆盖率

│   └── L2: 端外新客数
│       ├── L3: 广告 CPA
│       ├── L3: RTA 胜率

├── L1: 老客 MAO
│   ├── L2: 存量用户活跃率
│   │   ├── L3: D7/D30 留存率
```

### 5.2 典型使用场景

#### 场景 A：全链路增长方案（增长 PM）

```
输入: "帮我制定下周的货运增长方案，预算 50 万"

系统响应:
1. Orchestrator 识别意图 → scope=full
2. 并行分析:
   - ProspectAgent: 从 3,000 万出行用户中识别 32,000 名高潜用户，LTV 排序 Top 5,000
   - SubsidyAgent: 因果推断最优发券方案，¥10 券 ATE=0.12（显著），ROI 预估 3.2x
   - AdAgent: RTA 竞价优化，过滤规则调整后预计 CPA 降至 ¥55
3. ConversionAgent: 基于高潜用户分群设计 4 套触达方案（金刚位/Push/SMS/Banner）
4. RetentionAgent: 上周 3,200 新客养成进度 + 860 名高危用户预警
5. 汇总输出: KPI 看板 + 5 条策略建议 + 2 个实验推荐

预计完成时间: ~15s
```

#### 场景 B：补贴实验分析（数据分析师）

```
输入: "上周 10 元 vs 15 元满减券实验效果如何？"

系统响应:
1. Orchestrator → scope=subsidy
2. SubsidyAgent:
   - 实验回顾: 分组方式、样本量 12,000、运行 7 天
   - 因果推断: ATE=0.035 (p=0.012)，15 元券显著优于 10 元券
   - 价格弹性: 新用户 |弹性|=2.1（敏感），老用户 |弹性|=0.8（不敏感）
   - ROI: 15 元券 ROI=2.4x vs 10 元券 ROI=1.9x
   - 策略建议: 对新用户发 15 元券，老用户维持 10 元券
```

#### 场景 C：季节性运营准备（增长 PM）

```
输入: "毕业季搬家场景怎么准备？"

系统响应:
1. 识别季节性场景 → scope=full + seasonal_context=毕业季
2. ProspectAgent: 识别 18-25 岁、高校密集城市用户画像
3. ConversionAgent: 设计学生专享触达方案（社交媒体 Push + 校园 Banner）
4. SubsidyAgent: 学生群体价格弹性高 → 推荐大额首单券（¥20-30）
5. RetentionAgent: 去年毕业季新客 D30 留存仅 18% → 建议强化养成计划
6. AdAgent: 抖音学生群体定向投放 + 毕业季创意 A/B 测试
```

#### 场景 D：跨渠道预算优化（运营负责人）

```
输入: "下月预算 80 万，怎么在端内和端外分配？"

系统响应:
1. Orchestrator → scope=full
2. ProspectAgent: 各渠道高潜用户规模估算（端内 32,000 vs 端外预估 8,000）
3. SubsidyAgent: 端内补贴 ROI 3.2x vs 端外广告 CPA ¥80
4. AdAgent: RTA 出价优化后预计 CPA 降至 ¥55，推荐端外预算占比 25%
5. RetentionAgent: 端内获客 D30 留存 28% vs 端外 12%，端内用户长期价值更高
6. 汇总: 端内 65 万 / 端外 15 万，预留 5 万做实验，含 2 个实验推荐
```

## 六、竞品分析

### 6.1 竞品对比

| 维度 | 传统 BI (Tableau/Superset) | 规则引擎 | 纯 LLM 方案 | **GrowthPilot Agent** |
|------|---------------------------|---------|-------------|----------------------|
| 数据分析 | 静态报表 + 手动下钻 | 固定规则 + 阈值告警 | LLM 自由分析 | **ML 工具精确计算 + LLM 解读** |
| 策略生成 | 需人工解读数据后制定 | 预设策略模板 | LLM 直接生成 | **数据驱动 + LLM 策略生成 + 实验验证** |
| 因果推断 | 不支持 | 不支持 | 不支持（仅相关） | **DoWhy 因果推断** |
| 个性化 | 有限（分群报表） | 规则匹配 | LLM 生成 | **基于弹性/意向/LTV 的精确分层** |
| 实验平台 | 第三方集成 | 简单分流 | 无 | **内置实验设计 + 评估** |
| 全链路 | 需多个工具拼接 | 单点优化 | 单次对话 | **端到端全链路覆盖** |
| 成本 | License ¥50-100 万/年 | 开发维护成本高 | Token 成本不可控 | **多模型分层 + 开源框架** |
| 响应速度 | 分钟级（人工分析） | 秒级（规则触发） | 分钟级（多轮对话） | **秒级（单次执行 ~15s）** |

### 6.2 核心差异化优势

1. **计算精确性**：不同于纯 LLM 方案容易产生数值幻觉，GrowthPilot 用 Python 工具做精确计算（统计检验、优化求解），LLM 只负责解读和策略生成
2. **因果而非相关**：通过 DoWhy 因果推断框架，区分"补贴是否真的有用"和"用补贴的用户是否恰好更活跃"
3. **全链路闭环**：从潜客识别 → 转化 → 补贴 → 留存 → 投放，一个系统覆盖完整的用户生命周期
4. **渐进式降级**：每个环节都有 fallback 机制，单个工具失败不影响整体分析，系统鲁棒性强

## 七、商业模式

> 注：以下商业模式为面向 SaaS 化部署的规划方向，当前系统以核心引擎能力建设为主。

### 7.1 目标客户

- **Tier 1**：同城货运平台（货拉拉、滴滴货运、快狗打车）
- **Tier 2**：其他 O2O 平台（外卖、即时配送、社区团购）
- **Tier 3**：需要用户增长的传统企业（零售、金融、教育）

### 7.2 收费模式

| 模式 | 定价思路 | 适用场景 | 预估年收入 |
|------|---------|---------|-----------|
| **SaaS 订阅** | ¥5-20 万/月（按 MAO 规模分档） | 中大型平台 | 主要收入来源 |
| **按量计费** | ¥0.5-2/次分析调用 | 中小客户 / 灵活使用 | 补充收入 |
| **增值服务** | 定制化 Agent 开发 + 咨询 | 大客户 | 高毛利业务 |
| **数据服务** | 匿名化行业基准数据 | 行业分析需求 | 长尾收入 |

### 7.3 竞争壁垒

1. **行业 know-how**：沉淀了货运增长的领域知识（补贴策略、RTA 规则、留存节点）
2. **Agent 资产**：5 个专业 Agent 的工具和 prompt 可复用、可迁移
3. **数据飞轮**：更多客户 → 更多数据 → 更好的模型 → 更好的效果

## 八、风险与挑战

### 8.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **LLM 幻觉** | 高 | 中 | 精确计算由 Python 工具完成，LLM 仅做解读；结构化输出约束（Pydantic）|
| **数据质量** | 高 | 高 | 内置数据校验和降级机制；支持 demo 模式演示 |
| **LLM 调用成本** | 中 | 中 | 多模型分层调用（fast/default/power）；配置缓存减少重复调用 |
| **冷启动** | 中 | 高 | 提供合成数据 demo 模式；工具 fallback 到 heuristic |
| **延迟** | 低 | 中 | LangGraph 并行执行；多模型分层降低复杂任务延迟 |

### 8.2 业务风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **策略建议不当** | 中 | 高 | 人工审核机制；策略输出含置信度评分 |
| **市场竞争** | 高 | 中 | 持续迭代；聚焦垂直场景优势 |
| **客户需求变化** | 中 | 中 | Agent 架构灵活可扩展；Prompt 模板可快速调整 |
