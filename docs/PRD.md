# GrowthPilot Agent — 产品需求文档 (PRD)

> 版本：v5.0 | 日期：2026-04-21
> 状态：优化升级版

---

## 一、产品概述

### 1.1 产品定义

GrowthPilot Agent 是一个面向货运平台的 Multi-Agent 用户增长智能系统。覆盖用户全生命周期——潜客识别、端内转化、补贴优化、留存召回、域外投放——以实验驱动增长决策，用 ML 工具精确计算、LLM 解读并生成策略。

### 1.2 目标用户

| 用户画像 | 核心诉求 |
|---------|---------|
| 增长产品经理 | 快速获得增长策略建议和 A/B 实验方案 |
| 数据分析师 | 自动化分析流程，减少重复性数据处理 |
| 增长工程师 | 验证 ML 模型效果，优化补贴和投放策略 |
| 运营负责人 | 数据驱动的全局决策支持和预算审批 |

### 1.3 核心价值主张

1. **全链路覆盖**：从获客到留存，一个系统覆盖增长全生命周期
2. **实验驱动**：每个策略都通过 A/B 实验验证，而非拍脑袋
3. **精确计算 + AI 解读**：Python 工具精确计算指标，LLM 负责策略生成
4. **可解释性**：每一步分析都有推理过程，策略建议有数据支撑

## 二、功能需求

### 2.1 核心功能模块

#### M1: 编排引擎 (Orchestrator)

**FR-001 意图识别**
- 用户输入自然语言查询，系统自动识别分析范围
- 支持范围：full / prospect / conversion / subsidy / retention / ad / inapp
- 当前方案：LLM 意图理解（已实现）

**FR-002 任务调度**
- 根据意图调度对应的子 Agent 组合
- 支持并行执行（prospect / subsidy / ad 可并行）
- 支持顺序依赖（conversion 依赖 prospect + subsidy 结果）
- **当前方案**：LangGraph StateGraph 编排，prospect/subsidy/ad 真并行

**FR-003 结果聚合**
- 汇总各 Agent 分析结果，生成 KPI 快照
- LLM 综合分析，输出策略建议
- **优化需求**：增加执行进度反馈（事件流）
- **已实现**：Synthesis Node 综合分析 + 记忆系统上下文注入

#### M2: 潜客识别 (Prospect Agent)

**FR-010 特征工程**
- 从用户行为日志构建特征矩阵
- 支持自定义特征和自动特征选择

**FR-011 意向建模**
- LightGBM 模型预测用户货运转化意向
- 输出 AUC、Accuracy 等模型指标
- **优化需求**：Prompt 增加模型指标解读引导

**FR-012 用户评分与分群**
- 综合意向分数 + LTV 预测的加权评分
- RFM 分群 + 价值分群双维度分层

**FR-013 LTV 预测**
- BG/NBD + Gamma-Gamma 模型预测用户生命周期价值

#### M3: 端内转化 (Conversion Agent)

**FR-020 触达策略**
- 金刚位 / Banner / Push / SMS 多渠道触达方案
- 基于用户分群的差异化触达策略

**FR-021 漏斗分析**
- 曝光 → 点击 → 浏览 → 领券 → 下单 全链路转化分析
- 分群、分城市、分时段的漏斗对比

**FR-022 资源位分配**
- 基于流量和转化率的资源位优化
- 整数规划求解最优分配方案

**FR-023 券策略设计**
- 折扣券 / 满减券的面额和门槛设计
- 基于价格弹性的个性化发券

**FR-024 多触点归因**
- 支持首次触点 / 末次触点 / 线性 / 时间衰减 / U 型归因

#### M4: 补贴优化 (Subsidy Agent)

**FR-030 因果推断**
- DoWhy 框架分析补贴对转化的因果效应
- 控制混淆变量，得到真实的补贴效果

**FR-031 价格弹性**
- 估计不同用户分群的价格弹性系数
- 基于弹性设计差异化补贴方案

**FR-032 预算优化**
- 整数规划求解最优发券方案
- 约束条件：总预算、ROI 下限、公平性

**FR-033 ROI 计算**
- 实时计算补贴 ROI
- 与历史策略的 ROI 对比

#### M5: 留存召回 (Retention Agent)

**FR-040 新客养成**
- 首单后 7/14/30 天阶段性运营计划
- 每阶段的目标行为和触达策略

**FR-041 流失预警**
- 流失风险评分和分级（高/中/低风险）
- 流失原因分析

**FR-042 流失召回**
- 针对不同流失原因的召回策略
- 召回效果追踪

**FR-043 同期群分析**
- 按获客时间、渠道、城市维度的同期群分析
- 留存曲线对比

#### M6: 域外投放 (Ad Agent)

**FR-050 RTA 策略**
- 实时竞价策略设计
- 基于用户价值的出价策略

**FR-051 出价优化**
- OCPX 智能出价
- 分阶段出价策略（冷启动 → 稳定期）

**FR-052 创意分析**
- 广告素材效果评估
- 创意 A/B 测试建议

**FR-053 受众定向**
- Lookalike 受众扩展
- 基于用户画像的定向策略

#### M7: 实验平台

**FR-060 实验设计**
- 自动计算所需样本量（功效分析）
- 实验分组策略（随机、分层、聚类）

**FR-061 实验评估**
- t 检验、Mann-Whitney U 检验
- 序贯检验（边跑边看）
- ICE 优先级评分

## 三、非功能需求

### 3.1 性能需求

| 指标 | 当前 | 目标 |
|------|------|------|
| 全链路分析耗时 | ~30s (顺序执行) | ~12s (并行执行) |
| 单 Agent 分析耗时 | 5-10s | 5-10s (不变) |
| LLM 调用成本 | 100% | 40-60%（多模型分层已实现） |
| JSON 解析失败率 | ~15% | <5%（Pydantic 结构化输出已实现） |

### 3.2 可靠性需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| LLM 调用重试 | 3 次指数退避重试 | P1 |
| Provider 降级 | 主 Provider 失败自动切换备用 | P1 |
| 工具降级 | 单工具失败不影响整体分析 | P2 |
| 数据加载安全 | 路径校验、文件类型限制、大小限制 | P2 |

### 3.3 可观测性需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| 执行进度反馈 | SSE 实时推送各 Agent 执行状态 | P1 |
| 执行追踪 | 记录各 Agent 耗时、状态变化 | P3 |
| 质量自评 | Agent 输出质量评分 | P3 |

### 3.4 可扩展性需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| Hook 机制 | 已实现（PreRun/PostRun Hook） | ✅ |
| 工具注册表 | 动态注册和发现工具 | P2 |
| Prompt 模板 | Jinja2 模板化，支持继承和组合 | P3 |
| 持久化记忆 | 已实现（TF-IDF 语义搜索 + JSON 持久化） | ✅ |
| 结构化输出 | 已实现（Pydantic Models） | ✅ |

## 四、用户场景

### 场景 1: 全链路增长方案

```
输入: "帮我制定下周的货运增长方案，预算 50 万"

流程:
1. Orchestrator 识别意图 → scope=full
2. 并行执行:
   - ProspectAgent: 识别 3200 名高潜用户，含 LTV 排序
   - SubsidyAgent: 因果推断最优发券方案，ROI 预估 3.2x
   - AdAgent: RTA 竞价优化建议，预计 CPA 降至 ¥45
3. ConversionAgent: 基于潜客结果设计端内触达方案
4. RetentionAgent: 上周新客养成进度 + 流失预警
5. 汇总输出: KPI 看板 + 策略建议 + 实验推荐

输出: 结构化增长方案报告
```

### 场景 2: 补贴实验分析

```
输入: "上周 10 元 vs 15 元满减券 A/B 实验结果怎么样？"

流程:
1. Orchestrator 识别意图 → scope=subsidy
2. SubsidyAgent:
   - 实验回顾：分组方式、样本量、运行时长
   - 统计检验：t 检验 + 序贯检验
   - 效果评估： uplift、ROI、长期价值
   - 策略建议：基于弹性分析的最优券面额

输出: 实验分析报告 + 策略建议
```

### 场景 3: 季节性运营建议

```
输入: "马上毕业季了，搬家场景怎么准备？"

流程:
1. Orchestrator 识别意图 → scope=full (季节性)
2. 各 Agent 基于季节性上下文分析:
   - ProspectAgent: 识别大学生用户画像
   - ConversionAgent: 设计学生专享触达方案
   - SubsidyAgent: 学生群体价格弹性 → 优惠力度建议
   - RetentionAgent: 去年毕业季新客留存分析
   - AdAgent: 抖音/快手学生群体定向投放

输出: 季节性运营方案
```

## 五、成功指标

| 指标 | 衡量方式 | 目标 |

## 六、架构演进记录

| 版本 | 关键变化 |
|------|---------|
| v1.0 | 初始原型，关键词意图匹配 + 顺序执行，单 Agent 架构，纯 Prompt 驱动输出 |
| v2.0 | 引入多 Agent 分工（Prospect/Subsidy/Ad 等），伪并行调度，增加实验平台和归因模块 |
| v3.0 | LLM 意图理解替代关键词匹配，asyncio.gather 真并行，结构化 Prompt 降低 JSON 失败率 |
| v4.0 | LangGraph StateGraph 编排，PreRun/PostRun Hook 机制，多模型分层（重推理用强模型、轻任务用快模型） |
| v5.0 | Synthesis Node 综合分析，TF-IDF 语义搜索 + JSON 持久化记忆系统，Pydantic 结构化输出，全链路 ~12s 达成 |
